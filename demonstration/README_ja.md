
## 📁 Demonstration / Folder Overview

このフォルダは、Language OS（Mini Meaning OS）の **実証用デモ** を安全に公開するための構成です。
デモの UI と内部ロジックを明確に分離し、将来的な拡張時に「公開すべき部分」と「非公開にすべき部分」を安全に管理できるよう設計されています。

---

## 🧩 1. Hugging-Face-Spaces

ここには **Hugging Face 上で実行されるコード全体** を置きます。

Hugging Face は以下の 2 スペース構成を採用しているため、
対応するファイルをこのフォルダに整理しています：

### ① Public UI Space

* 一般公開されるデモ画面
* 外部から見えるのは UI と最小限の処理のみ
* **内部ロジック（モデル/CSV/コア処理）は含まない**
* Private Space を API 経由で呼び出すだけ
* `HF_TOKEN` を使って認証する（UI からは見えない）

### ② Private Backend Space

* Mini Meaning OS の本体
* core_concepts, meaning_triples, 解析ロジックなどすべて
* **非公開で安全に保持する**
* ここを公開すると内部データまで全て見えてしまうため隔離している
* Public UI はここに対して安全に API リクエストを送るだけ

---

## 📁 2. Public-UI-Space

Hugging Face の **Public UI Spaceと同じ構造のローカル版**。

* UI だけを管理するフォルダ
* 「公開してよい部分」だけを明確に切り分けている
* 今後、デモ UI の見た目や説明文を編集したいときはここだけ触ればよい

Public UI から「ユーザーの入力 → Backend API 呼び出し → 結果表示」
という形で内部処理が完全に分離されている。

---

## 🔒 分離構造を採用した理由（哲学 / 安全性）

この Language OS は今後：

* 意味ネットワークの拡張
* core_concept / triple の追加
* 一般ユーザーの参加 or 専門家レビュー
* 参考文献や evidence の取り扱い
* 将来の権利・ライセンスの整理

など、多くの点で **慎重に公開範囲を管理する必要がある**。

そのため、初期段階から以下の方針を定めている：

### ✔ UI（見せる部分）だけ公開

### ✔ 内部データ・ロジック（意味構造）は Private Space で秘匿

### ✔ 誤って公開してはいけない部分を物理的に分離

### ✔ 将来 “一部だけ公開 / 一部だけ限定共有” など柔軟な公開戦略が取れる

これにより、
**構造証明は公開できるが、内部データの権利問題や拡張リスクから守る**
というバランスが取れている。

## 🔧 今後の運用ポイント

* Public UI は「説明・デモ用」
* 内部データを改修するときは Private Backend Space のみを更新
* API 接続は HF_TOKEN によって安全に保護
* Token は公開されない（UI Space に見えない・コードにも書かない）
