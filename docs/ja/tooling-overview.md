## 開発優先度と安全ポリシー

本プロジェクトで想定している主なツール／応用領域は、便宜上つぎの 6 分野に整理する：

1. 対人コミュニケーション支援（会話・音声・態度制御）
2. 翻訳（テキスト / 音声 / 字幕）
3. 知識・研究支援（論文の構造化・概念検索）
4. 医療・生命科学領域での知識統合
5. 法律・契約領域での条件・解釈支援
6. 自立思考 AI（推論・メモリ・自己更新）

これらに対して、開発優先度と安全ポリシーを以下のように定める。

### 1〜3：当面の主要ターゲット（積極的に実装）

1. 対人コミュニケーション支援  
2. 翻訳  
3. 知識・研究支援  

これら 3 分野は、**当面の主な実装対象**とする。

- 意味三連結（meaning_triples）による構造化
- スロットビュー（WHO / WHAT / WHY / HOW / STATE ...）の生成
- 翻訳・要約・論文整理 UI など

は、この 1〜3 の範囲から優先して実装・検証を行う。

> 方針：  
> - 「誤情報」そのものを生成する機能ではなく、  
>   既存テキストの **意味構造化／比較／可視化** にフォーカスする。  
> - 出力はあくまで「入力テキストの構造化ビュー」であり、  
>   独立した事実主張を勝手に追加しない。

### 4〜5：専門家連携を前提とした慎重領域

4. 医療・生命科学  
5. 法律・契約  

これらの領域は、**人命・権利に直接関わる高リスク領域**であるため、

- 1〜3 の領域で十分な実績・安定性が確立された後に着手する。
- 必ず医療・法律などの **専門家と連携して設計・検証** を行う。
- 「100％、OS自身が誤情報を生成しない」形（※）でのみ実装する。

（※）ここでいう「誤情報を生成しない」とは：
- OS 自身が独自に診断・法律判断を下さず、
- 既存の正式な情報源（ガイドライン・判例・添付文書など）を  
  **構造化・参照・ナビゲーションするだけ** に限定する、という意味。

**したがって、** 4・5 の領域では、たとえば次のような制約つき実装を想定する：

- triple / conditions / evidence は **公的情報源の写像 + 専門家レビュー済みの内容** に限定
- 推論機能は「候補の列挙」「矛盾点のハイライト」までに留める
- 最終判断は、必ず人間の専門家に委ねる UI / API 設計とする

### 6：自立思考 AI（実験的・並行開発）

6. 自立思考 AI（推論・メモリ・自己更新）

この領域は、**他の分野と並行して独立に試行錯誤する実験的トラック**とする。

- OS 上に構造化された triple / evidence / stats を用いて、
  - 因果推論
  - 自己メモリの更新
  - 自律 triple 提案
  などを試行する。
- ただし、すぐに外部向け機能として公開するのではなく、  
  **内部検証とシミュレーション** を重ねたうえで、
  「実用レベルで安全に動作する」と判断できた場合のみ段階的に公開する。

> 方針：
> - 「自立思考 AI」は **研究・実験レイヤー** と明示し、  
>   1〜3（実用レイヤー）とは区別して扱う。
> - 意味OSの信頼性・透明性を損なわない範囲で、  
>   徐々に推論機能・自己更新機能を拡張していく。

---

このポリシーにより、

- 1〜3：実装と検証を進めやすい「攻め」の領域  
- 4〜5：専門家と連携しながら進める「慎重」領域  
- 6：研究として並行する「実験」領域  

という三層構造が明確になる。

## 1. 対人コミュニケーション（会話・音声・態度制御）

### 🔹 主な入力

* 音声（ASR）
* テキスト
* 相手の属性（友人 / 上司 / 不特定多数）
* シチュエーション（会議・SNS・ビジネス・ゲーム）

### 🔹 OSが担う処理

* UD + expr_links で文の意味を「triple 候補」に変換
* 9スロットにマッピングして **意図構造化**
* conditions によって **敬語 / フォーマル度 / 攻撃性抑制 / 感情表現** などを調整

→ **「どう伝えるべきか」を OS が構造的に判断できる**

---

## 2. 翻訳（テキスト / 字幕 / 同時通訳）

### 🔹 主な入力

* 原文（音声/テキスト）
* 翻訳目的（学術/字幕/チャット）

### 🔹 OSの役割

* 文を meaning_triple 化して構造的意味に落とす（lossless）
* core_concept の domain / region / era を参照して意味のズレを補正
* conditions_json に応じて丁寧さ / 表現長 /文化差 を処理

→ **「意味の保存」と「文化差の補正」が可能になる**

---

## 3. 研究・学術領域

* 定義 → triple
* 主張 → triple（肯定/否定）
* 根拠 → evidence
* 引用・再現性 → stats

→ **論文の構造化・概念検索・自動知識ベース化**

---

## 4. 医療・生命科学

* 機序（A → B を引き起こす）→ triple
* 副作用 → triple / evidence
* 禁忌 → conditions（年齢・併用薬・疾患）

→ **医療知識を条件付きの意味構造に落とせる**

---

## 5. 法律・契約

* 禁止 / 許可 → triple（正/負）
* 国 / 時代 / 制度 → conditions
* 判例 → evidence

→ **条文の意味を「条件付き命題」として扱える**

---

## 6. 自立思考AI（推論・メモリ）

* 内部ログ → triple化
* 推論結果 → evidence/stats で蓄積
* AI自身が意味構造を更新して成長

→ **OS が AI の「論理的な長期記憶」になる**
